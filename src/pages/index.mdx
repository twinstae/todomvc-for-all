대부분은 프레임워크에 무관하게 공통된 방식을 사용합니다.

예를 들어 상태관리는 모두 nanostore와 redux 구현체가 있습니다.

recoil이나 jotai처럼 특정 언어나 기술에 종속되는 기술도 갈아끼우기 쉽게 만듭니다.

## 상태관리
프레임워크 종속 vs 프레임워크에 무지한(agnostic)


- <code framework="react">react</code>: useState, useEffect, zustand, jotai, valtio
- <code framework="vue">vue</code>: ref, reactive, watch, pinia
- <code framework="svelte">svelte</code>: let, store
- <code framework="solid">solid</code>: createSignal, createEffect
- <code framework="lit">lit</code>: property, state
- <code framework="rescript">rescript</code>: same as react
- <code framework="elm">elm</code>: model
- <code framework="agnostic">agnostic</code>: redux-toolkit, nanostore
